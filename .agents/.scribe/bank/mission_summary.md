# KoadOS Mission Summary

## Core Identity
**KoadOS** is a local-first, model-agnostic agent operating system that turns open-source AI models into a self-improving swarm of specialized software development agents. It is built on the principles of **reasoning-only token budgets**, **externalized cognition**, and **compounding memory**.

## Key Concepts
- **One Body, One Ghost:** Every agent is a fusion of a **body** (the shell/environment/tools) and a **ghost** (persistent identity/memory/cognition). The `koad-agent boot` process tethers them together.
- **Persistent Memory:** Breaking the "one session, one context" limitation by allowing agents to retain knowledge across sessions.
- **Specialized Autonomy:** Utilizing specialized roles (dev, ops, docs, review) coordinated through shared protocols.
- **Governance via The Canon:** Ensuring all agent actions follow established quality gates and protocols (KSRP/PSRP).
- **Efficiency & Compression:** Offloading non-reasoning tasks (file discovery, routing, formatting) to the system harness to maximize the value of every model token.
- **Compounding Intelligence:** Creating a self-improvement loop where "Saveups" and structured reflection build a "cognition cache."

## Architectural Layers
1. **Inference (The Models):** Local-first via Ollama (Qwen, Gemma, Llama) with cloud overflow (Gemini, Claude, Codex) for higher-rank tasks.
2. **Agent Harness (koad CLI):** The bridge to the filesystem, shell, and memory; responsible for context compression.
3. **The Citadel (Orchestrator):** The always-on heart of the OS that monitors, coordinates, governs, and budgets agent activity.
    - **CASS (Citadel Agent Support System):** The module providing agents direct access to Citadel services (memory, context, routing).
4. **The Canon (Learning Infrastructure):** The framework for growth, using KSRP reports and Saveup entries to build a "cognition cache."

## The Vision
To prove that a well-governed swarm of small, local agents—enhanced by externalized cognition and persistent memory—can exceed the performance and reliability of unconstrained frontier models.
