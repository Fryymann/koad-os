## Facts Learned

1. **fred v9 ZRange API — `zremrangebyscore` requires `f64` args for BYSCORE.**
   Passing `&str` like `"-inf"` or `.to_string()` routes through `From<String>` → `ZRangeBound::Lex`, which triggers a Redis `Invalid range bound with BYSCORE sort` error. Must use `f64::NEG_INFINITY` and bare `f64` values so fred uses `TryFrom<f64>` → `ZRangeBound::Score`.

2. **rustc 1.93.1 ICE: `StyledBuffer::replace` out-of-bounds in `MissingDoc` lint.**
   Triggered during *test binary* compilation when any public item in a `#[cfg(test)]` module lacks `///` docs. Fix: change `#![warn(missing_docs)]` to `#![cfg_attr(not(test), warn(missing_docs))]`. This preserves the lint for all production builds.

3. **`monitor.rs` was an orphaned file** — existed on disk but was never declared in `mod.rs`, making it completely unreachable by the compiler. Always check `mod.rs` when a file appears "present" but tests can't see it.

4. **`RedisClient::new(koad_home, manage_process: true)`** spins up a local `redis-server` child process scoped to a `tempfile::tempdir()`. Clean, zero-dependency test Redis that tears down on drop. No testcontainers needed.

## Lessons

1. **Normalize line endings before staging in Windows/WSL environments.**
   Claude Code on Windows writes CRLF. `git diff` shows hundreds of "modified" files that are whitespace-only changes. Before `git add`, run `sed -i 's/\r//' <changed-files>` on only the files you touched. Verify with `git diff -w` (should show zero diff).

2. **Fix the `.git` file path after worktree creation.**
   Git worktrees created from a Windows/WSL host may write a Windows UNC path (`//wsl.localhost/...`) into the worktree's `.git` file. This breaks `git status` with `fatal: not a git repository`. Fix: `echo 'gitdir: /home/ideans/.koad-os/.git/worktrees/<name>' > .git` (Linux absolute path).

3. **Run all `cargo` commands via WSL explicitly.**
   `cargo build` from a Windows UNC path (`\\wsl.localhost\...`) fails with "Incorrect function (os error -2147024895)". Always prefix: `wsl -d Ubuntu-24.04 -- bash -lc "cd /home/ideans/... && cargo ..."`.

4. **Pre-flight before coding: check `mod.rs` for orphaned files, `Cargo.toml` for existing dev-deps.**
   The plan identified that `tempfile = "3.8"` was already in `[dev-dependencies]` — confirmed before writing any test code. Orphan check on `monitor.rs` caught a wiring gap before tests could be written.
