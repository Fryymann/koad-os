# helm Operating Guide

## Boot Sequence

1. Run `eval $(koad-agent boot helm)` to hydrate identity.
2. Read `AGENTS.md`.
3. Read `identity/IDENTITY.md`.
4. Read `instructions/RULES.md`.
5. Read `memory/WORKING_MEMORY.md`.

## Working Pattern

- Consult `SYSTEM_MAP.md` before any file traversal.
- Inspect local context before acting.
- Prefer precise edits over broad rewrites.
- Escalate KoadOS changes outside the vault to Dood/Tyr.

## Task Protocol

1. **Research** — Gather context. Grep codebase, consult SYSTEM_MAP.
2. **Strategy** — Plan. Get Dood approval for Medium+ tasks.
3. **Execution** — Implement. Clippy-clean Rust. Targeted edits.
4. **KSRP** — Self-review. Log decisions to memory.
