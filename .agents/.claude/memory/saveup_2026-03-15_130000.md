## Facts Learned (addendum to morning saveup)

5. **When rebasing after a file-move:** If upstream moved a file (e.g., `streams.rs` →
   `koad-core/src/signal.rs`), your tests on the old file become `DU` (delete/unresolved).
   Accept the deletion with `git rm <old-path>`, then port the tests to the new location.
   Check the new module's `Cargo.toml` for missing dev-deps before assuming tests will compile.

6. **Cross-crate test isolation:** `koad-core` cannot import from `koad-citadel`.
   When porting tests from citadel → core, replace `StreamMonitor` usage with direct
   fred calls (`xlen`, `xrange`) available via `StreamsInterface` which is already
   imported in `signal.rs`.

---

## Lessons (addendum)

5. **Check upstream PR activity before pushing.** If multiple PRs are open on the same
   codebase, a fast-moving team may merge structural refactors (file moves, crate
   reorganizations) that conflict with your work. Always `git fetch && git log --oneline
   origin/nightly -5` before the final push to anticipate rebases.
