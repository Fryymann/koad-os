# The KoadOS Development Canon

All tasks within KoadOS MUST follow this sequence:

1. **View & Assess**: Evaluate issue and system impact. Assign a task weight (`trivial`, `standard`, or `complex`) that governs KSRP scope.
2. **Brainstorm & Research**: Explore solutions and validate technical assumptions.
3. **Plan**: Create a detailed implementation map.
4. **Approval Gate (Dood)**: **STRICT HALT.** Agent must wait for explicit approval keywords (`Approved`, `Proceed`, `Go`).
5. **Implement**: Execute surgical code changes. **LOCKED** until Step 4 is verified.
6. **Koad Self-Review Protocol (KSRP)**: Execute the iterative review loop (lint, verify, inspect, architect, harden, optimize, testaudit).
7. **Reflection Ritual (PSRP)**: Execute the Post-Sprint Reflection Protocol: `Fact -> Learn -> Ponder`. Output is a Saveup Entry.
8. **Results Report**: Present code, KSRP Report, and Saveup Entry to Dood.
9. **Final Approval Gate (Dood)**: **STRICT HALT.** Agent must wait for explicit approval to close the issue.
